# PLAN

## Absolute Rule
- NO SUBAGENTS: all work for this project is done directly in this session.

## Objective
- Build a deep understanding of the refactored remora-v2 library (what it is, what it does, how it works), then perform a detailed review of its documentation for correctness, completeness, clarity, and consistency with implementation.

## Steps
1. Establish scope and inventory
- Map top-level package structure, key modules, and primary docs.
- Identify intended audience and likely usage paths from docs.

2. Understand library architecture from code
- Trace core abstractions, data flow, and lifecycle.
- Identify core invariants and extension points.
- Record implementation-backed behavior notes with file references.

3. Understand public usage from docs
- Read README and docs in user-facing order.
- Capture claims, examples, setup instructions, and guarantees.

4. Documentation review
- Check each major doc claim against code.
- Flag inaccuracies, stale content, missing prerequisites, ambiguity, and broken narrative flow.
- Classify findings by severity and affected audience.

5. Deliverables
- Produce detailed review report with prioritized findings and concrete fixes.
- Summarize strongest docs and highest-value improvements.

## Acceptance Criteria
- A concrete architectural understanding documented with references.
- Full documentation review with prioritized findings.
- Actionable recommendations tied to exact docs and implementation locations.
